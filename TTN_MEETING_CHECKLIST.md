# TTN meeting checklist

- Ask for official API documentation.
- Ask for sandbox access and test credentials.
- Ask for the authentication method: API key, OAuth2, mutual TLS, username/password, certificate, or another mechanism.
- Ask for the official TEIF version and XSD files.
- Ask for the required XML signature standard and profile.
- Ask whether signature is done before TTN submission or through a TTN-approved provider.
- Ask for submit invoice endpoint details.
- Ask for status polling endpoint details.
- Ask for proof/download endpoint details.
- Ask for error codes, rejection payloads, retry rules, and idempotency requirements.
- Ask for QR code and TTN reference generation rules.
- Ask for multi-client, accountant, or mandate requirements for SaaS platforms.
- Ask for legal archiving rules, retention duration, and accepted proof formats.
- Ask for rate limits, SLA expectations, and maintenance window notices.
